# 前端开发方案

## 1. 目标

构建一个知识型阅读工作台前端，支持文件上传、文件库、统一阅读器、划线批注、笔记编辑和 AI 阅读助手入口。

MVP 优先级：
1. 登录注册
2. 文件上传与文件库
3. PDF / Markdown / TXT / DOCX 阅读
4. 文本选中、高亮、批注
5. 笔记编辑与摘录
6. AI mock 面板

## 2. 技术栈

- Vue 3
- Vite
- TypeScript
- Vue Router
- Pinia
- Naive UI
- Tailwind CSS
- PDF.js
- Tiptap
- Markdown-it
- Axios

## 3. 页面规划

### `/login`
- 邮箱或用户名登录
- 密码输入
- 登录按钮
- 跳转注册页

### `/register`
- 用户名
- 邮箱
- 密码与确认密码
- 注册按钮

### `/dashboard`
- 顶部导航栏
- 左侧菜单
- 最近阅读
- 最近上传
- 我的批注
- 快速上传入口

### `/documents`
- 文件上传
- 文件列表
- 类型筛选
- 搜索
- 重命名
- 删除
- 进入阅读器

### `/reader/:documentId`
- 左侧：目录、缩略图占位、文件信息
- 中间：阅读区
- 右侧：批注、摘录笔记、AI 面板
- 顶部：返回、搜索、缩放、阅读模式、AI 总结、保存进度

### `/notes`
- 笔记列表
- 创建、编辑、删除
- 按文档筛选
- 按标签筛选

### `/settings`
- 用户资料
- 主题设置
- 存储空间
- AI API 配置预留

## 5. 核心模块

### 5.1 认证
- 登录、注册表单
- token 持久化
- 当前用户状态管理
- 路由守卫

### 5.2 文件库
- 上传组件
- 文件列表
- 搜索和筛选
- 重命名和软删除
- 文件详情页跳转

### 5.3 阅读器
- 通用 ReaderShell
- 依据文件类型切换渲染器
- PDF 使用 PDF.js
- Markdown / TXT 使用文本渲染
- DOCX 使用 HTML 占位预览

### 5.4 批注交互
- 文本选区识别
- SelectionToolbar
- 创建高亮、下划线、批注
- AnnotationLayer 渲染
- AnnotationSidebar 同步

### 5.5 笔记
- Tiptap 编辑器
- 笔记列表与筛选
- 从批注生成笔记

### 5.6 AI 面板
- 文档总结
- 解释选中文本
- 翻译选中文本
- 文档问答
- 先接 mock API

## 6. 状态管理

- `authStore`：登录态、当前用户、退出登录
- `documentStore`：文件列表、详情、上传进度
- `annotationStore`：批注列表、创建、删除、定位
- `noteStore`：笔记列表、编辑、筛选

## 7. 关键实现点

1. PDF 高亮必须使用相对坐标渲染。
2. Markdown / TXT 选区先保存 `selectedText` 和简化范围信息。
3. 组件拆分要清晰，阅读器逻辑不要堆在单个大组件里。
4. 请求统一通过 Axios 实例封装。
5. UI 保持工具型、克制、适合长时间阅读。

## 8. UI 原则

- 左侧导航固定
- 阅读器居中
- 右侧面板可折叠
- 顶部工具栏轻量
- 留白充足
- 高亮颜色柔和

建议色值：
- 背景：`#F7F7F5`
- 主文本：`#1F2937`
- 次文本：`#6B7280`
- 边框：`#E5E7EB`
- 主色：`#2563EB`

## 9. 前端开发顺序

1. 初始化 Vue 项目与基础布局
2. 接入路由、Pinia、请求层
3. 完成登录注册页
4. 完成文件库页
5. 完成阅读器基础壳
6. 完成 PDF / Markdown / TXT / DOCX 阅读
7. 完成文本选区与高亮
8. 完成批注侧边栏
9. 完成笔记模块
10. 接入 AI mock 面板

## 10. 验收点

- 能登录和注册
- 能看到上传文件
- 能进入阅读器
- PDF / Markdown / TXT / DOCX 可读
- 能高亮并添加批注
- 能创建和编辑笔记
- AI 面板能返回 mock 数据
