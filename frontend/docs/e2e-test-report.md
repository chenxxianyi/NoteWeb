---

## 失败根因总结

所有 15 个失败的测试案例（分布在 3 个测试文件中）都指向 **同一个根本原因**：

> **Go 后端上传文件后没有解析文件内容（`parsed_content` 字段始终为空）**，导致阅读器无法渲染文档正文，测试断言找不到对应的 DOM 元素而超时。

对于 PDF 文件还有第二个问题：
> **测试数据 `sample.pdf` 可能不是合法 PDF**，需要重新生成有效的测试 PDF。

---

## 修复建议

| 优先级 | 问题 | 建议修复 |
|--------|------|----------|
| P0 | 后端文件解析缺失 | 在 Go 后端的 Upload 流程中，上传文件后调用解析器（类似 Python 后端的 `FileParseService`），将解析结果存入 `parsed_content` |
| P1 | PDF 测试数据无效 | 用 `pdfkit` 或 Node.js 脚本来生成合法的测试 PDF |
| P2 | 测试稳定性 | 部分测试的上传等待时间 `waitForTimeout(3000)` 在压力下不够，可改为等待特定元素出现（`waitForSelector`） |
