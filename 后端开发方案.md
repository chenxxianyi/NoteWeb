# 后端开发方案

## 1. 目标

构建支撑文件阅读、批注、笔记和 AI 辅助能力的后端服务，负责鉴权、文件存储、元数据管理、内容解析、批注笔记存储以及统一 API 输出。

MVP 重点：
1. 用户认证
2. 文件上传与管理
3. 文档内容获取
4. 批注和笔记 CRUD
5. AI mock 接口
6. Docker Compose 本地运行

## 2. 技术栈

- Python 3.11+
- FastAPI
- SQLAlchemy 2.x
- Alembic
- MySQL 8
- Redis
- MinIO
- JWT
- Pydantic v2
- Uvicorn
- Celery
- Docker Compose

## 4. 核心职责拆分

### 4.1 `core`
- 配置管理
- JWT 与密码工具
- 依赖注入

### 4.2 `db`

- 软删除

## 7. 关键实现点

### 7.1 统一响应

成功：
```json
{ "code": 0, "message": "success", "data": {} }
```

失败：
```json
{ "code": 40001, "message": "错误信息", "data": null }
```

### 7.2 鉴权
- 使用 JWT
- 所有受保护接口通过 `Depends(get_current_user)`
- 用户只能访问自己的文档、批注和笔记

### 7.3 文件上传
- 校验大小
- 校验 MIME 和扩展名
- 生成唯一文件名
- 上传 MinIO
- 写入 documents 表

### 7.4 文件解析
- PDF：优先记录页数和元数据
- TXT：读取文本
- Markdown：读取原文
- DOCX：提取文本或 HTML 占位
- 图片：返回空解析内容

### 7.5 AI mock
- 先返回模拟结果
- 保留 provider 抽象
- 便于后续切换 OpenAI、Claude、Ollama 等

## 9. 开发顺序

1. 初始化 FastAPI 工程
2. 配置数据库与迁移
3. 实现注册、登录、JWT
4. 实现用户信息接口
5. 实现文件上传到 MinIO
6. 实现文件列表、详情、删除、内容接口
7. 实现批注 CRUD
8. 实现笔记 CRUD
9. 实现 AI mock 接口
10. 补充测试与 Swagger 校验

## 10. 验收点

- 可以注册、登录、获取当前用户
- 可以上传并查询文件
- 可以读取不同类型文档内容
- 可以创建、查看、删除批注
- 可以创建、编辑、删除笔记
- 可以调用 AI mock 接口
- Swagger 可访问 `/docs`
- Docker Compose 可本地启动
