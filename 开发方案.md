# 多格式文件阅读 + 划线批注 + 笔记编辑 + AI 阅读助手开发方案

## 1. 项目目标

构建一个知识型阅读工作台，支持 PDF、DOCX、Markdown、TXT、图片等文件上传、阅读、划线批注、笔记编辑和 AI 阅读辅助。  
MVP 阶段优先保证核心链路可运行：登录注册 -> 上传文件 -> 阅读文件 -> 高亮批注 -> 摘录笔记 -> AI mock 接口。

## 2. MVP 范围

### 必做
- 用户注册、登录、JWT 鉴权
- 文件上传、列表、详情、删除、重命名
- PDF / Markdown / TXT / DOCX 基础阅读
- 文本选择、高亮、批注、删除批注
- 笔记 CRUD、从批注生成笔记
- AI 总结 / 解释 / 翻译 / 问答 mock 接口
- Docker Compose 本地启动

### 暂缓
- OCR
- DOCX 精准版式还原
- AI 真实大模型接入
- 复杂全文检索
- 复杂多端同步

## 3. 技术方案

### 前端
- Vue 3 + Vite + TypeScript
- Vue Router + Pinia
- Naive UI + Tailwind CSS
- PDF.js 负责 PDF 渲染
- Tiptap 负责 Markdown / 富文本笔记编辑
- Markdown-it 负责 Markdown 解析
- Axios 统一请求封装

### 后端
- FastAPI + Uvicorn
- SQLAlchemy 2.x + Alembic
- MySQL 8 存结构化数据
- MinIO 存文件本体
- Redis 预留缓存和异步任务
- Celery 预留后台解析与 AI 任务
- Pydantic v2 负责入参和出参校验

## 4. 架构原则

1. 文件本体不进 MySQL，只进 MinIO。  
2. 批注、高亮、笔记都独立建表，不写回原文件。  
3. PDF 高亮使用相对坐标，避免缩放错位。  
4. 所有资源都必须做 user_id 权限校验。  
5. Endpoint 只收发请求，业务逻辑放 Service，数据库操作放 Repository。  
6. AI 先做 mock，但保留统一接口和 provider 抽象层。

- 文本选择与浮动工具栏

- 高亮层渲染

### 第六阶段：笔记系统
- 笔记 CRUD
- 从批注生成笔记
- 笔记列表与编辑器

### 第七阶段：AI 接口预留
- 文档总结 mock
- 选中文本解释 mock
- 翻译 mock
- 问答 mock
- 前端 AI 面板接入

### 第八阶段：优化与收尾
- 响应式布局
- 错误处理与空状态
- 加载态
- 基础测试
- 权限校验补强

## 6. 数据模型

### users
- 用户基础信息
- 密码哈希
- 存储使用量

### documents
- 文件元信息
- MinIO 存储路径
- 解析状态
- 解析结果
- 阅读进度

### annotations
- 文档、用户、页码
- 高亮类型、颜色
- 选中文本
- 坐标和范围信息

### notes
- 笔记内容
- 内容类型
- 关联文档和批注

## 8. 前端页面规划

- `/login`
- `/register`
- `/dashboard`
- `/documents`
- `/reader/:documentId`
- `/notes`
- `/settings`

阅读器页面采用三栏布局：
- 左侧：目录、缩略图、文件信息
- 中间：正文阅读区
- 右侧：批注、笔记、AI 面板

## 9. 后端目录落地

按以下职责拆分：
- `core`：配置、鉴权、异常
- `db`：数据库连接
- `models`：ORM 模型
- `schemas`：请求和响应结构
- `api`：路由层
- `services`：业务逻辑
- `repositories`：数据库访问
- `utils`：通用工具
- `tasks`：异步任务

## 10. 风险与处理

- DOCX 解析复杂：先做基础文本或占位 HTML。
- PDF 高亮定位：先保存相对坐标和基础范围数据。
- Markdown / TXT 选区恢复：先用 selectedText + 简化 rangeData。
- AI 接入不确定：先做 mock provider，再切换真实 provider。

## 11. 验收标准

- 能注册登录
- 能上传并看到文件
- 能进入阅读器
- PDF / Markdown / TXT / DOCX 可读
- 能高亮、批注、删除批注
- 能摘录为笔记并编辑
- 能保存阅读进度
- AI 面板可返回 mock 结果
- Docker Compose 可一键启动
- Swagger 可访问 `/docs`

## 12. 推荐执行顺序

1. 初始化前后端工程和 Docker
2. 做认证闭环
3. 做文件上传和文件库
4. 做阅读器和内容解析
5. 做高亮批注
6. 做笔记系统
7. 做 AI mock 接口
8. 做体验和稳定性优化
