# Requesting Code Review 中文说明

## 这个 skill 是什么

用于在完成重要开发任务后请求代码评审，帮助尽早发现问题、回归和遗漏。

## 什么时候使用

- 完成一个主要功能后
- 准备合并前
- 完成复杂 bug 修复后
- 重构前后需要确认风险

## 怎么使用

1. 确定本次改动范围。
2. 获取 base 和 head 的 Git SHA。
3. 让代码评审子代理只关注这段 diff。
4. 按严重程度处理评审意见。

## 注意事项

- 评审上下文要精确，不要给无关历史。
- Critical 和 Important 问题应优先修复。
- Minor 问题可以记录后续处理。
