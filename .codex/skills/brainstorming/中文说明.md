# Brainstorming 中文说明

## 这个 skill 是什么

用于在动手实现前，把一个模糊想法整理成清晰设计方案。它强调先理解需求、提问、比较方案、获得用户确认，再进入实现。

## 什么时候使用

- 开发新功能前
- 修改已有行为前
- 设计组件、页面、交互流程前
- 需求还不够清楚，需要和用户一起澄清时

## 怎么使用

1. 先阅读项目上下文。
2. 一次只问一个关键问题。
3. 给出 2 到 3 种可选方案和取舍。
4. 整理成设计说明并让用户确认。
5. 用户确认后再进入实施计划。

## 注意事项

- 不应在设计获得确认前直接写代码。
- 简单任务也要至少形成一个简短设计判断。
- 适合创意、产品、交互、功能设计类工作。
