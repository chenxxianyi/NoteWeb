# Using Git Worktrees 中文说明

## 这个 skill 是什么

用于在开始较大功能开发前建立隔离工作区，避免影响当前分支或用户已有改动。

## 什么时候使用

- 开发新功能需要隔离环境
- 当前工作区已有未完成改动
- 要执行一份较大的实施计划
- 需要多个分支并行工作

## 怎么使用

1. 先检测当前是否已经在 worktree 中。
2. 如果不是，询问是否需要创建隔离 worktree。
3. 优先使用平台原生隔离能力。
4. 没有原生能力时再使用 `git worktree`。

## 注意事项

- 不要在已经隔离的 worktree 里再创建一层。
- 创建 worktree 前要尊重用户偏好。
- 子模块场景需要额外判断。
