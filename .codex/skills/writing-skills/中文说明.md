# Writing Skills 中文说明

## 这个 skill 是什么

用于创建、修改和验证新的 skill。它把 TDD 思想应用到流程文档写作：先设计压力测试场景，再写 skill，并验证代理是否真的按 skill 行动。

## 什么时候使用

- 创建新的 Codex / Claude skill
- 修改已有 skill
- 验证 skill 是否有效
- 想沉淀可复用工作流程时

## 怎么使用

1. 先设计没有 skill 时容易失败的压力场景。
2. 验证基线行为确实会失败。
3. 编写最小有效的 `SKILL.md`。
4. 再次验证代理是否按要求执行。
5. 根据新漏洞继续改进 skill。

## 注意事项

- skill 应该是可复用方法，不是一次性复盘故事。
- 没有验证过的规则可能只是愿望。
- 写 skill 前应理解 TDD 的红绿重构循环。
