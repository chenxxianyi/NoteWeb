# Dispatching Parallel Agents 中文说明

## 这个 skill 是什么

用于把多个互不依赖的问题分派给多个子代理并行处理，以节省时间并减少主会话上下文负担。

## 什么时候使用

- 多个测试文件失败，且原因彼此独立
- 多个模块有独立问题
- 多个调查任务可以并行完成
- 每个任务不需要共享状态

## 怎么使用

1. 判断任务是否真的相互独立。
2. 为每个问题域创建一个独立子任务。
3. 给每个子代理明确范围、输入和期望输出。
4. 汇总子代理结果，再决定最终修改方案。

## 注意事项

- 相关问题不要强行拆开。
- 子代理不应继承完整会话历史，只给必要上下文。
- 有共享文件或顺序依赖时应谨慎并行。
