# Subagent-Driven Development 中文说明

## 这个 skill 是什么

用于按实施计划进行开发时，把每个相对独立的任务交给新的子代理执行，并在每个任务后做规格符合性和代码质量评审。

## 什么时候使用

- 已经有明确实施计划
- 计划中的任务相对独立
- 当前平台支持子代理
- 希望减少上下文污染并提高并行质量

## 怎么使用

1. 读取实施计划。
2. 为每个任务创建独立子代理。
3. 子代理完成后先做规格评审。
4. 再做代码质量评审。
5. 主会话负责整合和推进。

## 注意事项

- 紧密耦合任务不适合强行拆分。
- 子代理指令必须包含足够上下文。
- 不要在每个小任务之间反复询问用户是否继续。
